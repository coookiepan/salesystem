## 這是什麼？

DUSKIN 銷售系統是一個專為 **外勤業務人員** 設計的行動網頁應用程式（PWA）。

它可以幫你：

- **管理客戶資料** — 記錄客戶名稱、地址、聯絡人、狀態（未拜訪、試用中、已成約...）
- **記錄拜訪日誌** — 每次拜訪的筆記、心情、照片
- **追蹤試用品項** — 哪些客戶正在試用什麼商品、報價多少
- **管理車上庫存** — 即時掌握拖把、地墊、芳香劑的庫存數量
- **產出日報** — 自動彙整每天的拜訪紀錄
- **待辦清單** — 所有客戶的待辦事項一覽
- **店家查詢** — 搜尋公司現有客戶名單

### 特色

- 📱 **手機優先設計** — 在手機上操作像原生 App 一樣順暢
- 🔌 **離線可用** — 沒有網路也能正常使用，資料存在你的手機裡
- ☁️ **雲端備份** — 連接 Google Sheets 後，資料會同步到雲端，不怕手機資料遺失
- 🆓 **完全免費** — 使用 GitHub Pages 免費部署，不需要付任何費用

## Step 1：Fork GitHub 專案

> 「Fork」就是把別人的專案複製一份到你自己的帳號底下。這樣你就有了一個屬於自己的版本。

### 1.1 打開原始專案

用電腦瀏覽器打開這個網址：

👉 **https://github.com/coookiepan/salesystem**

### 1.2 點擊 Fork

你會看到頁面右上角有一個 **「Fork」** 按鈕（旁邊通常有一個數字）。

1. 點擊 **Fork**
2. 跳出的頁面中，「Owner」（擁有者）選你自己的帳號
3. 「Repository name」保持預設的 `salesystem` 就好
4. 點擊綠色的 **「Create fork」** 按鈕

等幾秒鐘，你會看到頁面跳轉到 **你自己帳號下** 的 `salesystem` 專案。網址會變成 `https://github.com/你的帳號名稱/salesystem`。

### 1.3 開啟 GitHub Pages

現在要讓這個專案變成一個可以打開的網頁：

1. 在你 Fork 後的專案頁面，點上方的 **「Settings」**（設定，齒輪圖示）
2. 左邊選單往下找到 **「Pages」**，點擊它
3. 你會看到「Build and deployment」區塊
4. 在 **「Source」** 那一行，點下拉選單，選擇 **「Deploy from a branch」**
5. 在 **「Branch」** 那一行，把 `None` 改成 **`main`**，旁邊的資料夾選 **`/ (root)`**
6. 點擊 **「Save」**

### 1.4 等待部署完成

GitHub 會花大約 1~3 分鐘來部署你的網站。部署完成後：

- 回到 **Settings → Pages** 頁面
- 你會在上方看到一個綠色方框，寫著：「Your site is live at...」
- 你的網站網址就是：

```
https://你的帳號名稱.github.io/salesystem/
```

用手機瀏覽器打開這個網址，你會看到 DUSKIN 銷售系統的畫面！🎉

> 💡 **小提示**：第一次可能需要等幾分鐘才能打開。如果顯示 404，請耐心等待，重新整理頁面試試。

## Step 2：建立 Google 試算表

> Google 試算表會作為你的「雲端資料庫」，所有客戶資料都會備份到這裡。

### 2.1 打開 Google Sheets

用瀏覽器打開：

👉 **https://sheets.google.com**

如果還沒登入 Google，系統會請你登入。

### 2.2 建立空白試算表

1. 點擊左上角的 **「+」**（空白試算表），或點擊「空白」的方格
2. 你會看到一個新的空白試算表被建立出來

### 2.3 幫試算表取個名字

1. 左上角會顯示「未命名的試算表」
2. 點擊它，改成一個好辨識的名字
3. 按 Enter 確認

> ⚠ **重要：命名規則（多人團隊必看）**
>
> 如果你的公司會有多位業務員共用這套系統，**請把試算表檔名直接設為你的姓名**（例如：「王大明」、「陳小華」）。
>
> 原因：長官端的儀表板會自動讀取每個業務員試算表的「整份檔名」當作業務員身份，並把該業務員的試用金額、成約金額等指標顯示出來。檔名取錯，儀表板上會顯示錯誤的業務員名稱。
>
> ✅ 建議：`王大明`
> ✅ 建議：`陳小華 - 銷售系統`（檔名前段是姓名即可）
> ❌ 避免：`銷售系統`、`未命名的試算表`、`Untitled spreadsheet`

你不需要在試算表裡手動建立任何工作表（sheet）。稍後 App 連線成功並「全部推送」後，系統會自動幫你建立需要的工作表，包括：`clients`（客戶）、`inventory`（庫存）、`products`（商品）、`shops`（店家）。

> 💡 **記住**：先不要關掉這個試算表頁面，接下來的步驟需要用到它。

## Step 3：設定 Google Apps Script

> Apps Script 是 Google 提供的程式碼執行平台。我們需要在試算表裡放一段程式碼，讓 App 能跟試算表溝通。你不需要懂程式碼，只要複製貼上就好。

### 3.1 打開 Apps Script 編輯器

1. 在你剛才建立的 Google 試算表中
2. 點上方選單的 **「擴充功能」**（Extensions）
3. 在下拉選單中點 **「Apps Script」**

你會看到一個新的分頁被打開，這就是 Apps Script 的程式碼編輯器。畫面上會有一些預設的程式碼（通常是 `function myFunction() { }` 之類的）。

### 3.2 從 App 複製程式碼

1. 用瀏覽器打開你的 DUSKIN 銷售系統（就是 Step 1 建好的那個網址）
2. 點底部選單的 **「設定」**（齒輪圖示，最右邊的按鈕）
3. 往下滾動，你會看到「設定步驟」區塊
4. 找到最下方的按鈕：**「複製 Apps Script 程式碼」**
5. 點擊它，系統會顯示 **「✅ 已複製 Apps Script 程式碼」**

程式碼已經複製到你的剪貼簿了！

### 3.3 貼入 Apps Script 編輯器

1. 切回到 Apps Script 編輯器的分頁
2. **全選**編輯器裡現有的所有程式碼（按 `Ctrl+A`，Mac 使用者按 `Cmd+A`）
3. **刪除**它們（按 `Delete` 或 `Backspace`）
4. **貼上**剛才複製的程式碼（按 `Ctrl+V`，Mac 使用者按 `Cmd+V`）

你會看到編輯器裡出現一大段新的程式碼。

### 3.4 儲存

按 `Ctrl+S`（Mac 使用者按 `Cmd+S`），或點選上方工具列的 💾 儲存圖示。

你會看到上方顯示「已儲存專案」的提示。

## Step 4：部署 Apps Script 為網頁應用程式

> 「部署」就是把程式碼發佈到網路上，讓你的 App 能透過一個網址來存取它。

### 4.1 開始部署

1. 在 Apps Script 編輯器中，點右上角的藍色 **「部署」** 按鈕（Deploy）
2. 選擇 **「新增部署作業」**（New deployment）

### 4.2 設定部署選項

你會看到一個彈出視窗：

1. 左上角「選取類型」旁邊有一個 ⚙️ 齒輪圖示，點擊它
2. 選擇 **「網頁應用程式」**（Web app）

接著填寫以下設定：

| 欄位 | 填什麼 |
|------|--------|
| **說明**（Description） | 隨便填，例如 `v1` |
| **執行身分**（Execute as） | 選擇 **「我」**（Me） |
| **誰可以存取**（Who has access） | 選擇 **「任何人」**（Anyone） |

3. 點擊 **「部署」**（Deploy）

### 4.3 授權（重要！第一次才需要）

因為這段程式碼需要存取你的 Google 試算表，Google 會請你授權：

1. 點 **「授權存取」**（Authorize access）
2. 選擇你的 **Google 帳號**
3. 你可能會看到一個警告畫面，寫著「Google hasn't verified this app」（Google 尚未驗證此應用程式）—— 這是正常的！因為這是你自己寫的（貼的）程式碼。
4. 點左下角的 **「Advanced」**（進階）
5. 點 **「Go to [你的專案名稱] (unsafe)」**（前往...（不安全））
6. 點 **「Allow」**（允許）

> ⚠️ 這個步驟很多人會卡住。看到「不安全」不用緊張，這只是因為 Google 沒有審核過你自己的程式碼而已，完全安全。

### 4.4 複製部署網址

授權完成後，你會看到一個「部署完成」的畫面，上面有一段 **Web App URL**，長得像這樣：

```
https://script.google.com/macros/s/AKfycbx..........很長一串.........../exec
```

**這段網址非常重要！** 請點旁邊的 **「複製」** 按鈕把它複製下來。

> 💡 建議把這段網址先貼到記事本或 LINE 的自己的聊天室，以免搞丟。

## Step 5：連接 App 與 Google Sheets

> 現在回到你的 DUSKIN 銷售系統 App，把 Google Sheets 跟 App 連起來。

### 5.1 貼上網址

1. 打開你的 DUSKIN 銷售系統（`https://你的帳號名稱.github.io/salesystem/`）
2. 點底部選單的 **「設定」**（最右邊的齒輪圖示）
3. 找到 **「Google Sheets 同步」** 區塊
4. 在 **「Apps Script Web App URL」** 的輸入框裡，貼上你剛才複製的那段 URL
5. 系統會自動儲存這個網址

### 5.2 測試連線

1. 點 **「測試連線」** 按鈕
2. 等幾秒鐘，你會看到上方出現一個綠色提示框，顯示連線成功的訊息

如果連線成功，代表 App 已經能跟你的 Google 試算表溝通了！

如果顯示錯誤，請參考下方的[常見問題排解](#常見問題排解)。

### 5.3 推送資料到雲端

1. 點 **「全部推送」** 按鈕
2. 確認對話框點「確定」
3. 等待推送完成

完成後，你可以切回 Google 試算表看看，會發現自動新增了好幾個工作表（`clients`、`inventory`、`products` 等），裡面就是你的資料！

🎊 **恭喜！設定完成！** 你現在可以開始使用 DUSKIN 銷售系統了。

## Step 6：（選用）店家查詢資料

> 「查詢」功能可以讓你搜尋公司現有的客戶名單。如果你手上有這份名單，可以把它放進 Google 試算表裡。

### 6.1 找到 shops 工作表

1. 打開你的 Google 試算表
2. 在完成「全部推送」之後，你會在底部看到一個叫做 **`shops`** 的工作表標籤
3. 點擊它切換過去

### 6.2 填入店家資料

`shops` 工作表的欄位如下：

| 欄位 A | 欄位 B | 欄位 C | 欄位 D |
|--------|--------|--------|--------|
| 顧客名 | 電話號碼 | 地址 | 負責人 |

你可以手動逐筆輸入，也可以用複製貼上的方式把已有的名單貼進去。

> 💡 **小提示**：如果你手上有 Excel 格式的名單，可以直接把對應的欄位複製貼上到 Google Sheets 裡。

### 6.3 在 App 中載入

1. 回到 DUSKIN 銷售系統 App
2. 點底部選單的 **「查詢」**（放大鏡圖示）
3. 點右上角的 **「🔄 重新載入」** 按鈕
4. 等待載入完成，你就可以用搜尋框搜尋店名、地址、電話、負責人了

## 功能總覽

以下簡單介紹每個功能頁面的用途：

### 👥 客戶（底部選單第 1 個）

這是你最常用的頁面。

- **客戶列表**：所有客戶一覽，可用狀態分類篩選（未拜訪、初訪、複訪、試用中、報價、將成約、已成約、已轉交、拒絕）
- **搜尋**：輸入關鍵字即時搜尋客戶
- **排序**：依試用日、下次拜訪日、狀態、最後更新、名稱排列
- **新增客戶**：點「+ 新增」按鈕，填入基本資料（名稱、地址、類型、狀態...）
- **客戶詳情**：點擊任一客戶可查看完整資訊、拜訪記錄、試用品項
- **記錄拜訪**：在客戶詳情頁面新增拜訪紀錄（包含日期、筆記、心情）
- **試用品項**：追蹤客戶正在試用哪些商品、報價金額

### 📅 日報（底部選單第 2 個）

- 選擇日期，查看當天所有的拜訪記錄
- 自動彙整統計數據（拜訪數、新客戶數等）
- 方便回顧和製作日報

### ✅ 待辦（底部選單第 3 個）

- 集中顯示所有客戶的待辦事項
- 勾選已完成的項目
- 可篩選未完成 / 已完成

### 🔍 查詢（底部選單第 4 個）

- 搜尋公司現有客戶名單（資料來自 Google Sheets 的 `shops` 工作表）
- 輸入店名、地址、電話、負責人都能搜尋到
- 點「重新載入」可從雲端取得最新資料

### 📦 庫存（底部選單第 5 個）

- 管理你車上的庫存數量
- 分類瀏覽：地墊、拖把、芳香
- **+1 / -1**：快速調整庫存數量
- **缺貨清單**：一鍵查看哪些品項需要補貨
- 搜尋品項代號

### ⚙️ 設定（底部選單第 6 個）

- **商品資料庫**：管理所有商品（代號、品名、單價、計費週期）
- **版本資訊**：查看目前使用的版本號，點「強制重新載入」更新到最新版
- **Google Sheets 同步**：設定連線、測試連線、推送 / 拉取資料
- **資料管理**：匯出 JSON 備份 / 從 JSON 備份匯入

## 重要注意事項

### 🔄 Apps Script 程式碼更新

當 DUSKIN 銷售系統更新版本時，有時候 Apps Script 的程式碼也會跟著更新。如果你看到 App 提示需要更新後端程式碼，請依照以下步驟操作：

1. 在 App 的「設定」頁面，點 **「複製 Apps Script 程式碼」** 取得最新程式碼
2. 打開 Google Apps Script 編輯器（Google 試算表 → 擴充功能 → Apps Script）
3. 全選舊程式碼 → 刪除 → 貼上新程式碼 → 儲存
4. 點 **「部署」** → **「管理部署作業」**（Manage deployments）
5. 點右邊的 ✏️ 編輯圖示
6. 在「版本」（Version）那裡，把它改成 **「新版本」**（New version）
7. 點 **「部署」**

> ⚠️ 這邊容易出錯的地方：一定要選「新版本」，不要選舊的版本號。否則新程式碼不會生效。

### 💾 資料儲存方式

- App 的資料主要存在瀏覽器的 **localStorage** 裡（就是你手機/電腦瀏覽器的本地儲存空間）
- 如果你清除瀏覽器資料、清除快取、或換一台手機，本機資料就會消失
- 所以一定要設定好 Google Sheets 連線，定期「全部推送」來備份資料
- 萬一本機資料遺失，可以用「從雲端拉取」把資料從 Google Sheets 拉回來

### 📴 離線使用

- App 在沒有網路的狀況下也能正常運作
- 你可以繼續新增客戶、記錄拜訪、調整庫存
- 等有網路時，再去「設定」裡按「全部推送」同步到雲端

### 📱 加到手機桌面

讓 App 用起來像原生應用程式一樣方便：

**iPhone（Safari）：**
1. 用 Safari 打開你的 App 網址
2. 點底部的「分享」按鈕（方框+向上箭頭的圖示）
3. 往下找到 **「加入主畫面」**
4. 點「新增」

**Android（Chrome）：**
1. 用 Chrome 打開你的 App 網址
2. 點右上角的「三個點」選單
3. 選擇 **「加到主畫面」** 或 **「安裝應用程式」**
4. 點「新增」

加到桌面後，你會在桌面上看到一個 App 圖示，點開就直接進入銷售系統，跟一般 App 一樣。

### 📲 多裝置使用

如果你想在手機和電腦上都使用，或者換了新手機：

1. 在每一台裝置上，都打開同一個 App 網址
2. 在「設定」裡都貼上 **同一個** Google Sheets Web App URL
3. 在新裝置上，點 **「從雲端拉取」** 就可以把所有資料同步過來

> 💡 注意：如果兩台裝置都有修改資料，記得先在一台推送，再到另一台拉取，避免資料衝突。

## 常見問題排解

### ❌ 「測試連線失敗」

可能原因與解決方法：

1. **URL 貼錯了**  
   → 確認你貼的是完整的 Web App URL，開頭是 `https://script.google.com/macros/s/` 結尾是 `/exec`

2. **Apps Script 還沒部署**  
   → 回到 Apps Script 編輯器，確認已經執行了「部署 → 新增部署作業」的步驟

3. **沒有授權**  
   → 第一次部署時需要授權。如果跳過了，請重新部署一次

4. **權限設定不對**  
   → 確認部署時「誰可以存取」設定為 **「任何人」**

### ❌ 「版本過舊」

- 打開 App 的「設定」頁面
- 點 **「🔄 強制重新載入」** 按鈕
- 這會清除瀏覽器快取並重新載入最新版本

### ❌ 「資料消失了」

如果你發現客戶資料不見了，可能是瀏覽器的 localStorage 被清除了（例如清除瀏覽紀錄時一併清除了）。

**解決方法：**
1. 打開 App →「設定」
2. 確認 Apps Script Web App URL 還在（如果也被清掉，重新貼上）
3. 點 **「從雲端拉取」**
4. 系統會從 Google Sheets 下載所有資料回來

> 前提是你之前有成功「全部推送」過。如果從來沒推送過，Google Sheets 裡就沒有資料可以拉取。

### ❌ Apps Script 顯示「Script function not found」

- 回到 Apps Script 編輯器
- 確認你已經貼上了 **完整的** 程式碼
- 確認沒有只複製到一半
- 確認已按 `Ctrl+S` 儲存
- 重新部署：部署 → 管理部署作業 → 編輯 → 版本選「新版本」→ 部署

### ❌ 打開網頁顯示 404 Not Found

- 確認你已經在 GitHub 的 Settings → Pages 裡啟用了 GitHub Pages
- 確認 Branch 選的是 `main`
- 部署需要 1~3 分鐘，請稍等後重新整理頁面
- 確認網址格式正確：`https://你的帳號名稱.github.io/salesystem/`

### ❌ Google 授權畫面顯示「This app is blocked」

這通常發生在使用公司的 Google Workspace 帳號時。解決方法：
- 改用個人的 Gmail 帳號來建立試算表和 Apps Script
- 或聯繫你的公司 IT 管理員開放權限

## 長官儀表板（多業務員彙總用）

如果公司有多位業務員，長官（主管）想看所有業務員的銷售數字，使用這個功能。

### 啟用方式

長官在原本的 CRM 網址後面加上 `?admin=1`：

```
https://你的帳號名稱.github.io/salesystem/?admin=1
```

建議**把這個網址加進書籤**。一般業務員的網址不要加 `?admin=1`，他們不會看到長官按鈕。

### 設定步驟

1. 用 `?admin=1` 網址開啟 CRM
2. 右上角點版本號 → 出現黃色按鈕「📊 長官儀表板」
3. 在彈出的視窗中：
   - 把每位業務員的 Apps Script URL **一行一個**貼進文字框
   - 按「儲存清單」
   - 按「載入彙總」
4. 自動顯示每位業務員的：
   - **試用軒數**（狀態為試用中／報價／將成約的店家數）
   - **試用金額**（試用品項的 4W 原價加總，公司預設 2W 換週）
   - **成約軒數**（狀態為已成約的店家數）
   - **成約金額**（成約品項的 4W 報價加總，若無報價則用原價）
   - **成約轉換率**（成約軒數 ÷（試用軒數 ＋ 成約軒數））
5. 最上方顯示**所有業務員的總計**

### 長官如何取得業務員的 URL？

請業務員到自己的 CRM → 設定 → Google Sheets 同步區塊，把上面那條長長的 URL 複製給長官即可。每位業務員的 URL 都不一樣。

### 注意事項

- 此儀表板**唯讀**，不會修改任何業務員的資料
- 數字依據業務員當下雲端的資料，建議業務員每天開工前按「從雲端拉最新」確保資料同步
- 若某業務員試算表有未清理的重複店家，儀表板會顯示黃色警告（請業務員到「清理重複店家」處理）
